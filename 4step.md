이제 마지막 폴리싱 단계야! 지금까지 완성한 최고 밀도의 텍스트와 레이아웃 위에, 마치 실리콘밸리 하이테크 스타트업이나 Awwwards 수상 웹사이트 같은 세련된 '인터랙션과 마이크로 애니메이션'을 구현해 줄 차례야. 라이브러리 없이 순수 바닐라 자바스크립트의 Intersection Observer API와 CSS transition을 활용해 매우 가볍고 부드럽게 작동하도록 설계해 줘.

[필수 인터랙션 세부 가이드]
1. 스크롤 트리거 페이드업 (Scroll Reveal):
   - 사용자가 스크롤을 내릴 때, 프로필 정보 카드와 각 프로젝트의 메타데이터 블록, 상세 본문 블록들이 화면에 진입하는 순간 아래에서 위로 25px 정도 부드럽게 떠오르며 나타나게(opacity 0 -> 1, y 25px -> 0) 자바스크립트로 구현해 줘.
   - 한 섹션 내의 여러 하위 요소들은 0.1초의 시차(Stagger delay)를 두고 순차적으로 떠오르게 연출해 줘.
2. 프로젝트 카드 호버 이펙트 (Hover micro-interaction):
   - 프로젝트 카드 영역 전체에 마우스를 올리면, 카드의 테두리(border) 선 색상이 포인트 컬러로 은은하게 그라데이션되거나 변하게 하고, 아주 미세하게 위로 떠오르는(translate-y-[-4px]) 뎁스 효과를 선사해 줘.
   - GNB의 Contact 버튼이나 푸터의 이메일 전송 버튼에 마우스를 올리면 아이콘이 오른쪽으로 스르륵 움직이거나 배경색이 채워지는 마이크로 인터랙션을 넣어 줘.
3. 스크롤 진행 바 (Scroll Progress Bar):
   - 사용자가 긴 포트폴리오를 읽는 도중 자신의 위치를 쉽게 가늠할 수 있도록, 화면 최상단 GNB 바로 아래에 스크롤 높이에 따라 늘어나는 아주 얇고 감각적인 포인트 컬러 프로그레스 바를 구현해 줘.
4. GNB 스크롤 반응성:
   - 첫 화면(Hero)의 최상단 영역에 머물 때는 GNB의 배경이 완전히 투명했다가, 스크롤을 아주 조금이라도 내리면 불투명한 글래스모피즘 레이어(backdrop-blur-md bg-opacity-80)가 부드럽게 활성화되도록 자바스크립트를 작성해 줘.

[모바일 반응형 최적화 검수 규칙]
- 데스크탑에서 보여주던 화려한 '좌우 2단 분할(Sticky) 에디토리얼 레이아웃'이 모바일 디바이스(md 미만 해상도)에서는 자동으로 풀리며 '순차적인 상하 1단 레이아웃(flex-col)'으로 완벽하게 전환되도록 검수해 줘.
- 모바일 환경에서는 Sticky 속성이 비활성화되고 자연스러운 자연 흐름(static/relative)으로 대체되어 텍스트가 겹치거나 잘리지 않고 스크롤되도록 보장해 줘.
- 패딩과 폰트 크기가 모바일 화면 너비에 맞게 유동적으로 작아지도록 Tailwind의 유동적인 유틸리티 클래스를 아낌없이 적용해 줘.

절대로 중간에 생략이나 주석 처리를 하지 말고, `<html>`부터 `</html>`까지 한 번에 복사해서 바로 띄울 수 있는 최종 'index.html' 전체 단일 코드를 끝까지 끊김 없이 출력해 줘.
