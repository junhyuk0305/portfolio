# 🎒 강의 1주일 전 준비 안내

> RoundTable AI 하이엔드 웹 포트폴리오 마스터클래스 수강생용 사전 안내입니다.
> 강의 1주일 전에 이 문서를 받아 보시고, 강의 당일에는 **이미 모든 환경이 준비된 상태**로 오시면 됩니다.
> 준비 안 되셔도 강의 시작 시 1:1로 도와드립니다.

---

## ✅ 1. 필수 준비물 (4가지)

### 1) 노트북
- **OS**: Windows 또는 Mac (둘 다 가능)
- **브라우저**: **Chrome** (필수). Safari·Edge도 작동하지만 GitHub Pages 배포 단계의 모바일 미리보기에서 Chrome DevTools가 가장 안정적입니다.
- 충전기·마우스 지참 권장 (3시간 진행).

### 2) 구글 계정
- Gemini AI Studio 접속용입니다.
- 이미 Gmail을 쓰고 계시면 그대로 사용. 새로 만드실 분은 `https://accounts.google.com` 가입.
- **요금**: 무료. 신용카드 등록 불필요.

### 3) GitHub 계정 ⭐ 가장 중요
- 강의 마지막 단계(웹 배포)에 사용합니다.
- `https://github.com` 접속 → 우측 상단 Sign up → 이메일·비밀번호·**사용자명** 입력 후 가입.
- ⚠️ **사용자명(Username)이 곧 여러분의 사이트 URL이 됩니다.**
  - 예: 사용자명이 `taeyoonkim`이면 → 최종 사이트 URL은 `https://taeyoonkim.github.io`
  - 너무 길거나 회사·이력서에 적기 부끄러운 이름은 피하세요.
  - 추천 형식: `본인이름 + portfolio` 또는 `본인이름 + dev` (예: `taeyoonkim-portfolio`)
- 가입 후 이메일 인증까지 반드시 완료.

### 4) 500자 자기소개 + 포트폴리오용 경험 정리
- 자소서: 본인 정체성·강점·관점이 담긴 500자 내외 한 단락.
- 경험 정리: 프로젝트 1~3건 (각 250~400자, 줄글로 정리되어 있어도 OK).
- **포맷 무관**: Word, 한글, PDF, 노션, 구글 독스, 메모장 다 됩니다. 강의 중 텍스트만 복사해서 붙여넣으면 됩니다.

### 🆘 자소서·경험이 없으시다면
강사가 견본 4종을 미리 준비합니다.
- PM 지망생 김태윤 (RoomSync MVP 런칭)
- 마케터 지망생 박서연 (릴스 바이럴 + Meta Ads)
- HR 지망생 최민호 (ATS 도입 + 신입 온보딩)
- 프론트엔드 지망생 정수민 (React 스터디 플래너 + Web Speech 해커톤)

강의 시작 시 안내해 드리고, 본인 결이 맞는 한 명 선택해서 그대로 따라하시면 됩니다.

## 🔍 2. 강의 시작 5분 전 환경 점검 (체크리스트)

강의장 도착 후 강사가 시작하기 전 아래 5가지만 확인해 주세요.

### ✅ Gemini AI Studio
1. `https://aistudio.google.com` 접속
2. 좌측 상단 'Sign in' → 본인 구글 계정 로그인
3. 좌측 메뉴에서 **Create Prompt** (또는 **Chat**) 클릭
4. 채팅창과 우측 패널이 보이면 OK

### ✅ GitHub
1. `https://github.com` 접속 → 로그인
2. 우측 상단 **+** 아이콘 클릭 → 드롭다운에 **New repository** 메뉴가 보이는지 확인
3. 보이면 OK (실제로 Repository를 만드는 건 강의 중에 합니다)

### 🛠 Gemini AI Studio 권장 설정 (선택 — 강의 중 함께 합니다)
강의 중 다 같이 설정하지만, 미리 해두시면 좋습니다.

- 우측 패널 'Model': **최신 Gemini Pro 계열** (예: `gemini-2.5-pro` 또는 그 시점의 최상위 모델)
- 우측 패널 'Output length': **최대치**로 슬라이드 끌어올리기 → 단일 HTML 코드가 길어 잘리는 사고 예방
- 우측 패널 'Safety settings': 모두 **Block none**으로 → 코드 검열로 중간이 잘리는 사고 예방
- 'Temperature': **0.7 ~ 1.0** 권장 (너무 낮으면 카피가 딱딱함)

## ⚠️ 4. 강의 중 가장 자주 막히는 지점 3가지

미리 알아두시면 강의장에서 당황하지 않으실 수 있습니다.

### 🔴 1) AI 출력이 중간에 끊긴다
**증상**: Gemini AI Studio가 단일 HTML 코드를 생성하다가 `</main>` 직전 같은 곳에서 갑자기 멈춤.

**원인**: Output length 한도에 걸렸거나, 모델이 한 번에 출력 가능한 토큰을 다 썼음.

**대응**: 채팅창에 한 줄만 입력하세요.
```
이어서 작성해 줘
```
끊긴 지점부터 정확히 이어서 출력됩니다.

### 🟡 2) GitHub Pages가 404를 띄운다
**증상**: 배포 직후 URL 접속해 보니 "404 Not Found"가 뜸.

**원인**: GitHub Pages가 사이트를 빌드하는 데 1~2분 걸립니다. 너무 빨리 접속한 것.

**대응**: 1~2분 대기 후 새로고침. 그래도 안 되면:
- Repository 이름이 정확히 **`(사용자명).github.io`** 형식인지 확인
- 업로드한 파일명이 정확히 **`index.html`** 인지 확인 (대소문자 구분, `Index.html` X)

### 🟢 3) 모바일에서 화면이 깨진다
**증상**: 데스크탑에서는 멀쩡한데 핸드폰에서 보니 Hero 카피가 잘리거나 카드가 화면 밖으로 튀어나옴.

**대응**: 같은 Gemini AI Studio 세션에 한 줄 추가:
```
모바일 360px 이하에서 깨지는 부분을 고쳐 줘.
Hero 카피 폰트 한 단계 축소, 좌우 패딩 16px로 축소.
```

## 🔄 5. 강의 후 사이트 유지보수 가이드

오늘 만든 사이트는 강의가 끝나도 계속 살아 있습니다. 수정 방법 4가지를 미리 알려드립니다.

### 텍스트 수정
- 가장 간단한 방법은 GitHub 웹 에디터입니다.
- `(사용자명).github.io` Repository → `index.html` 파일 클릭 → 우측 상단 ✏️ 연필 아이콘 → 직접 텍스트 수정 → 하단 **Commit changes** 클릭
- 1분 후 라이브 사이트에 자동 반영

### 디자인 수정 (컬러·레이아웃 등)
- Gemini AI Studio의 같은 세션에서 이어서 부분 수정 프롬프트를 던지세요.
- 예: `[E] Impact 섹션의 큰 숫자 색을 #2563EB로 바꿔 줘`
- 새로 받은 HTML 전체를 GitHub `index.html`에 복사·교체

### 프로젝트 추가
- 4번째 프로젝트가 생겼다면 Gemini에 통째로 요청:
- 예: `프로젝트 4를 추가해 줘. 다음 7블록을 사용. [A] ... [G] ...`
- 자동으로 메인 페이지 4번째 카드 + #project-4 상세 페이지가 생성됨

### 도메인 연결 (선택)
- `taeyoonkim.com` 같은 본인 도메인을 쓰고 싶으면:
  1. 가비아·후이즈 등에서 도메인 구매
  2. GitHub Repository → Settings → Pages → Custom domain
  3. 구매처 DNS에 GitHub Pages IP 등록 (구매처마다 가이드 다름)
- 강의에서는 다루지 않습니다. 필요하시면 강의 후 강사에게 문의.
