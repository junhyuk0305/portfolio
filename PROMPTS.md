# 메가 프롬프트 5종 — 강의 현장 복사 시트

> 강의 진행 순서대로 `①` → `②` → `③` → `④` → `⑤`를 동일 Gemini AI Studio 세션에 순차 입력하세요.
> 5번 프롬프트가 끝나면 단일 `index.html` 한 파일이 손에 들어옵니다.
>
> **핵심 목표**: "원하는 디자인의 인터랙티브 웹 포트폴리오". 그래서 ②번 단계가 새로 추가되었습니다.
> 본인이 좋아하는 포트폴리오 사이트(awwwards, godly.website, siteinspire 등)에서 전체 캡쳐를
> 한 장 뜬 뒤, AI에게 던지면 → 우리 25종 갤러리 중 가장 가까운 1종 + 캡쳐의 디테일을 합친
> "디자인 디테일 카드"가 떨어집니다. 그 카드를 마지막 ⑤에 통째로 부어넣는 흐름입니다.

---

## 🧭 사용 흐름 한눈에

| 단계 | 입력 | 출력 | 다음 단계 입력으로 |
|---|---|---|---|
| **①** | 500자 자기소개 | 7블록 브랜드 키 (페르소나·카피·비주얼 디렉션 카드) | ②의 브랜드 키 입력 |
| **②** ⭐ NEW | 레퍼런스 사이트 전체 캡쳐 1장 + ①의 비주얼 디렉션 카드 + design.md 25종 | "디자인 디테일 카드" 1장 (25종 중 1종 매칭 + 캡쳐의 디테일 3~5개 + 컬러·폰트 확정) | ⑤의 [A] 슬롯에 통째로 임베드 |
| **③** | 경험 원본 줄글 1~3건 | STAR 4블록 ×3 | ④ 입력 |
| **④** | STAR 결과 | 본문 7블록 (A~G) ×3 | ⑤ 본문 데이터로 |
| **⑤** | ②의 디자인 디테일 카드 + ①의 Hero 카피 + ④의 본문 7블록 ×3 | 단일 `index.html` | GitHub Pages 업로드 |

> 💡 **②번 단계가 캡쳐를 안 받는 경우 (캡쳐 없이 25종 중 골라서 진행)**: 그래도 ②를 그대로 돌리되,
> "캡쳐 이미지 없음" 이라고 명시. 25종 중 1종 매칭 + ①의 비주얼 디렉션 카드만으로 디자인 디테일
> 카드를 채워서 출력합니다.

## 프롬프트 ① — 자소서 → 브랜드 키 + 비주얼 디렉션 카드

**언제 쓰는가**: 강의 SLIDE 12. 본인 500자 자기소개를 던지면 카피·페르소나·비주얼 디렉션이 모두 정리된 7블록 브랜드 키가 나옵니다.

**무엇을 강화했는가**: 6번 '비주얼 디렉션 카드'에 비주얼 패밀리·컬러 분위기·타이포 무드·정보 밀도·모션 결·이미지 의존도·첫인상 7개 축을 명시하도록 했습니다.
- 다음 ②번 단계에서 이 카드가 "캡쳐 사이트가 내 브랜드와 충돌하지 않는지" 정합성 체크의 기준이 됩니다.
- 컬러 Hex나 폰트 이름 같은 픽셀 단위 결정은 ②번에서 캡쳐와 함께 확정합니다(과도한 사전 결정으로 디자인 자유도를 죽이지 않기 위함).

```text
너는 시니어 브랜드 스트래티지스트이자 아트 디렉터야.
내가 제공하는 500자 자기소개를 분석해서, 다음 단계에서
(a) 레퍼런스 캡쳐 사이트가 내 브랜드와 충돌하지 않는지 검증하는 '기준'으로 쓰일 수 있고,
(b) 캡쳐가 없더라도 25종 갤러리에서 1종을 매칭할 만큼
방향성이 분명한 브랜드 키 한 장을 작성해줘.

---
[입력: 나의 500자 자기소개]
(여기에 자기소개 전문을 통째로 붙여넣으세요)
---

[출력 양식 — 정확히 7개 블록. 번호와 순서를 그대로 지킬 것.]

1. 페르소나 한 줄 (40자 이내)
   - 이 사람이 한 줄로 이렇게 기억되고 싶은 정체성

2. 브랜드 무드 키워드 3개
   - 서로 결이 다른 형용사 3개 (예: 분석적인 / 침착한 / 구조적인)
   - 셋 중 하나는 반드시 '감각계' 형용사(차가운/따뜻한/거친/정제된/묵직한 등)를 포함

3. Hero 메인 카피 후보 3개 (각 15자 이내)
   - 자소서의 핵심 정체성을 한 줄로 응축
   - 후보별로 어떤 결을 강조했는지 짧게 명시

4. Hero 서브 카피 후보 2개 (각 2줄 이내)
   - 메인 카피를 보강하며 가치 제안을 풀어낸 문장

5. 퍼스널 태그 3개 (해시태그 형식)
   - 직무 + 기술 + 결을 압축
   - 예: #데이터기반_PM #API명세_가능 #스크럼_리드

6. 비주얼 디렉션 카드 (7개 축, 한 줄씩 + 자소서 근거 1줄)
   - 6-1 비주얼 패밀리: 아래 7가지 중 1개 선택
     · 미니멀·뉴트럴 / 에디토리얼 매거진 / 다크·테크 /
       브루탈리스트·실험 / 컬러풀·플레이풀 /
       럭셔리·세리프 시그니처 / 모션·인터랙션 강조
   - 6-2 컬러 분위기: 아래 6가지 중 1개 선택
     · 모노크롬(흑백 한정) / 따뜻한 뉴트럴(베이지·크림 계열) /
       차가운 뉴트럴(그레이·블루그레이 계열) / 비비드 컬러(채도 높은 다색) /
       다크 + 단일 액센트 / 럭셔리(블랙+골드 또는 크림+버건디)
   - 6-3 타이포 무드: 아래 5가지 중 1개 선택
     · 정밀한 산세리프(엔지니어링·시스템) / 클래식 세리프(저널·권위) /
       모노스페이스 테크(코드·터미널) / 디스플레이 세리프 시그니처(럭셔리·패션) /
       그로테스크 견고함(산업·모더니즘)
   - 6-4 정보 밀도: 여백 압도형(60%+ 여백) / 균형형 / 정보 밀도형 중 1개
   - 6-5 모션 결: 정적·고요 / 미세한 페이드만 / 시네마틱·스크롤 연출 /
                인터랙티브·반응형 중 1개
   - 6-6 이미지 의존도: 텍스트 중심(이미지 0~1장) / 사진·이미지 중심 /
                      그래픽·도형·일러스트 중심 중 1개
   - 6-7 첫인상 한 줄: 사이트를 처음 연 사람이 3초 안에 받아야 할 인상을
                    형용사+명사 한 줄로 (예: "정돈된 연구실", "따뜻한 편집숍",
                    "조용한 갤러리", "정밀한 콘솔")

7. (다음 단계 ②번 매칭용 힌트) 25종 중 어울릴 만한 후보 3종 (선택)
   - 25종 갤러리(design.md)가 이번 세션에 로드되어 있을 때만 작성.
   - 각 후보: [번호 + 이름 + 1순위/2순위/3순위] + 근거 1줄.
   - 갤러리가 세션에 없으면 "해당 없음 — 다음 ②번에서 캡쳐와 함께 매칭"으로 기재.

---
[작성 규칙]
- 6번 블록은 제시된 카테고리 라벨 안에서만 고를 것. 임의 신조어 금지.
  단, 각 축마다 "자소서의 어떤 문장/근거에서 도출했는가"를 1줄로 함께 적을 것.
- 컬러 Hex, 정확한 폰트 이름, 픽셀 단위 레이아웃은 적지 말 것 (그 결정은 ②번 책임).
  6번 블록의 '카테고리 라벨' 수준 방향성까지만 명시할 것.
- 자소서에 없는 정보를 임의로 만들지 말 것. 비주얼 디렉션도 자소서의 결에서 유추한
  근거가 있어야 적을 수 있음.
- 모든 출력은 한국어, 비즈니스 톤.
- 출력 양식의 번호와 순서를 그대로 유지할 것.
```

## 프롬프트 ② — 레퍼런스 캡쳐 + 25종 갤러리 → 디자인 디테일 카드 ⭐ NEW

**언제 쓰는가**: 강의 SLIDE 15. 강의의 핵심 단계입니다.

**원칙**: "원하는 디자인을 손에 든다" — 본인이 awwwards, godly.website, siteinspire, 또는 평소에 봐둔
포트폴리오 사이트에서 마음에 드는 사이트를 1개 골라 **전체 페이지를 캡쳐**합니다.
그 캡쳐 한 장이 디자인 출처가 되고, 우리 25종 갤러리는 "검증된 베이스" 역할을 합니다.
AI가 캡쳐를 분석해서 → 25종 중 가장 가까운 1종을 베이스로 삼고 → 캡쳐의 디테일 3~5개를 덧붙여
"디자인 디테일 카드 1장"으로 합성합니다.

**캡쳐 방법 (강의 시연용)**:
- Chrome DevTools: `F12` → `Cmd/Ctrl + Shift + P` → "Capture full size screenshot"
- 또는 Chrome 확장 "GoFullPage" 사용
- 해상도가 길어도 OK. 전체 사이트의 톤이 한 장에 담기는 게 핵심.

**캡쳐가 없는 경우**: "캡쳐 이미지 없음" 이라고 입력. ①의 비주얼 디렉션 카드만으로 25종 중 1종을
고르고, 디자인 디테일 카드를 채웁니다(원래의 매뉴얼 갤러리 선택 흐름과 동일).

```text
너는 시니어 아트 디렉터이자 디자인 시스템 엔지니어야.
나는 인터랙티브 웹 포트폴리오를 만들고 있고, '원하는 디자인'을 두 가지 소스에서 합성하려고 해.
   (A) 내가 좋아하는 포트폴리오 사이트의 전체 캡쳐 한 장 (있을 수도 / 없을 수도 있음)
   (B) 강사가 제공한 검증된 25종 디자인 갤러리 (design.md)

너의 일은 (A)와 (B)를 하나의 "디자인 디테일 카드"로 합성하는 것이다.
이 카드는 다음 ⑤번 프롬프트의 [A] 슬롯에 그대로 임베드되어, 단일 index.html의
컬러·폰트·레이아웃·모션 결을 100% 결정한다.

---
[입력 1: 캡쳐 이미지]
(여기에 레퍼런스 사이트의 전체 페이지 캡쳐를 업로드.
 없으면 "캡쳐 이미지 없음" 이라고만 적기)

[입력 2: 프롬프트 ① 출력의 6번 비주얼 디렉션 카드]
(7개 축이 적힌 카드 전문을 그대로 붙여넣기)

[입력 3: 25종 갤러리]
(design.md의 25종이 이번 세션에 이미 로드되어 있다고 가정.
 안 되어 있으면 "design.md 갤러리 로드 필요" 라고 출력하고 중단할 것)
---

[출력 양식 — 정확히 5개 블록. 번호와 순서를 그대로 유지할 것.]

1. 캡쳐 사이트 분석 (캡쳐가 있을 때만 — 없으면 "캡쳐 없음, ①의 비주얼 디렉션 카드로 대체")
   - 1-1 첫인상 한 줄 (형용사+명사, 예: "조용한 갤러리")
   - 1-2 비주얼 패밀리 1개 (①과 동일한 7개 라벨 중)
   - 1-3 컬러 분위기 1개 + 캡쳐에서 추출한 주요 Hex 3~5개 (배경 / 본문 / 액센트 / 보조)
   - 1-4 타이포 무드 1개 + 캡쳐에서 추정한 영문 폰트 페어링 2개
     (Google Fonts에서 매칭되는 폰트명 — 예: Inter + Instrument Serif)
   - 1-5 레이아웃 그리드 (1단/2단/3단/비대칭/풀블리드 등) + 여백 비율 추정
   - 1-6 모션·인터랙션 신호 (정적 / 페이드 / 시네마틱 스크롤 / 마우스 인터랙티브 등)
   - 1-7 이미지 의존도 (텍스트 중심 / 사진 중심 / 그래픽 중심)

2. 25종 갤러리 매칭 — 최적 1종
   - 번호 + 이름 (예: "#01 Whitespace Cathedral")
   - 이 1종을 고른 이유: 캡쳐 분석의 어떤 축 3개가 일치하는지 명시
     (캡쳐가 없으면 ①의 비주얼 디렉션 카드의 어떤 축 3개가 일치하는지)
   - 차순위 1개 (2순위 번호 + 이름 + 1줄 근거) — 학생이 1순위가 마음에 안 들 때의 대안

3. 캡쳐에서 가져올 디테일 3~5개 (25종 베이스에 덧붙일 보강 명세)
   - 각 디테일은 "어디에 / 무엇을 / 왜" 3요소로 작성
   - 예: "Hero 섹션 / 메인 카피 우측에 회전하는 모노스페이스 메타데이터 라벨 추가 /
          캡쳐 사이트가 좌측 카피 + 우측 사이드 미터로 '엔지니어링 신호'를 강화하기 때문"
   - 디테일은 컬러 / 폰트 페어링 / 레이아웃 보정 / 모션 추가 / 마이크로 인터랙션 중에서
   - 캡쳐가 없으면 "캡쳐 디테일 없음. 25종 베이스 그대로 사용" 으로 1줄 작성

4. 브랜드 키 정합성 체크
   - 캡쳐(또는 매칭된 25종)와 ①의 비주얼 디렉션 카드 사이에 충돌하는 축이 있는지 검사
   - 충돌이 있다면 어느 축이 / 어떻게 어긋나는지 / 어느 쪽을 우선해야 하는지 1줄 권고
   - 충돌이 없으면 "충돌 없음 — 캡쳐 디테일을 그대로 채택" 으로 1줄 작성

5. 최종 디자인 디테일 카드 (⑤번 프롬프트 [A] 슬롯에 통째로 붙여넣을 형태)
   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   ▸ 베이스 디자인: [25종 중 번호 + 이름]
   ▸ 적용해야 하는 25종 베이스 프롬프트의 핵심 규칙: (3~5줄로 요약)
   ▸ 컬러 팔레트 (Hex 확정):
     · 배경: #______
     · 본문 텍스트: #______
     · 액센트(주): #______
     · 액센트(보조): #______ (있을 때만)
   ▸ 폰트 페어링 (Google Fonts 이름 확정):
     · 헤드라인: ______
     · 본문: ______
   ▸ 레이아웃 결: (1줄)
   ▸ 모션 결: (1줄)
```
