period: 2023.07 → 2023.11 · role: Service Planner / Scrum Master · team: PM 1 + BE 2

$ identify -- team.jpg · roomsync_kickoff · 16:9 · OK

학내 50여 개 동아리가 공용 회의실을 수기 예약하여 겹침 사고가 빈번한 문제를 해결하기 위해 기획·런칭한 웹서비스입니다. 수요 발굴 → MVP 추출 → 백엔드 협업 → 4개월 런칭까지 풀사이클을 주도했습니다.

$
## 문제 — problem

학내 50여 개 동아리가 공용 회의실을 단톡 공지 / 수기 장부로 예약하다 보니, 같은 시간대 겹침과 노쇼가 반복되고 있었습니다.

  • 예약 충돌이 빈번 // 회장단 정성 인터뷰 기준
  • 노쇼로 빈 방을 다른 동아리가 쓰지 못하는 자원 손실
  • "누가 언제 쓰는지 모른다"는 공통 페인포인트
진짜 병목은 "예약 기능 부재"가 아니라 "실시간 가시성"의 부재였다.
$
## 리서치 — research

회장단 대상 설문과 1:1 인터뷰로 니즈를 정량/정성 양면에서 검증했습니다.

  • 회장단 설문 — 핵심 니즈 = 실시간 현황 확인
  • 회장단 인터뷰 — 운영 맥락 / 페인포인트 청취
  • 기존 대안(노션 캘린더, 카톡봇 등) 비교 분석

→ "예약을 잘 만드는 것"보다 "현황을 본다"가 더 큰 가치임을 확인.

## MVP 결정 — scope

BE 2명 · 4개월이라는 리소스 제약 안에서, 검증된 핵심 니즈만 1차에 남기는 결단을 내렸습니다.

  • [IN] 예약 / 취소 / 실시간 룸뷰 — must-have
  • [IN] 회장단 권한 분리 — basic auth
  • [OUT] 푸시 알림, 통계 대시보드, 결제 → 2차 스프린트로 보류
  • [OUT] 외부 캘린더 sync → 3차로 후순위
"하지 않을 것" 리스트가 길수록, "할 것"의 정확도가 올라간다.
$
## 실행 — execution

개발 지식을 살려 API 명세 초안을 직접 작성해 백엔드 팀과의 소통 비용을 줄였습니다.

  • Jira 도입 · 스크럼 운영 · 스프린트 단위 진행
  • Figma 와이어프레임 · 팀 합동 리뷰
  • API 명세 초안 작성 · BE 2명과 정렬
  • 회장단 피드백 루프 — 가설 → 검증 반복
