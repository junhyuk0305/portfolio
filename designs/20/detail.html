RoomSync — 학내 동아리방 예약 관리

학내 50여 동아리가 공용 회의실을 수기로 예약하다 보니 시간 겹침이 잦았다. 회장단 설문으로 '실시간 현황' 니즈를 확인하고, 예약·취소·실시간 뷰 3가지만 갖춘 MVP를 1개월에 출시한 웹서비스.

IMG_INTERVIEW · 리서치
PHASE 01 / 문제 정의

수기 예약 → 시간 겹침

50여 동아리가 공용 회의실을 칠판·단톡으로 예약하다 겹침이 잦았다. 회장단 구글폼 설문에서 '실시간 현황'이 가장 큰 니즈로 드러났다.

  • 회장단 구글폼 설문 (니즈 우선순위)
  • 학내 50여 동아리 사용 패턴 관찰
  • 핵심 가설: 실시간 현황만 있어도 겹침이 사라진다
IMG_LECTURE · 실행
PHASE 02 / 실행

MVP = 예약·취소·실시간 뷰

가설 검증에 필요 없는 알림은 2차로 미루고, 3가지 핵심만 1개월에 출시. 서비스 기획자로서 API 명세 초안을 작성하고 Jira 스크럼 마스터를 맡았다.

  • MVP 스코프 3기능 확정 (알림은 2차)
  • API 명세 초안 · Notion 기반 문서화
  • Jira 백로그 · 스크럼 마스터 · Agile
IMG_PORTRAIT · 결과
PHASE 03 / 결과

30개 동아리 · 노쇼 -60%

출시 1개월 만에 30개 동아리가 가입했고, 회장단 운영 회의 기준 노쇼율이 60% 줄었다. 알림 기능을 2차 우선순위로 확정.

  • 30개 동아리 가입 (50여 중)
  • 노쇼율 60% 감소
  • 출시 1개월 · 2차 백로그 정의 완료
