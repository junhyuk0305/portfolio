데이터로 병목을 끊는 PM, 김태윤

비즈니스의 목적과 개발의 언어를 동시에 이해하는 서비스 기획자. 가설은 데이터로, 검증은 현장의 목소리로 — 프로덕트의 병목을 끊어내는 풀사이클 PM이 등장했다.

By EDITORIAL DESK · Seoul 2026.05.22 · 06:00 KST

김태윤(경영학·컴퓨터공학 복수전공)은 가설 수립부터 API 명세 작성, 스프린트 관리까지 풀사이클 기획 역량을 다져온 PM이다. 그는 "비즈니스의 목적과 개발의 언어를 동시에 이해하며 프로덕트의 병목을 끊어내는 기획자"라고 자신을 소개한다. 데이터로 가설을 세우고, 사용자의 목소리로 그 가설을 검증하는 일을 두려워하지 않는다는 점이 그의 작업 방식의 핵심이다.

대표작 'RoomSync'는 학내 50여 개 동아리가 수기로 회의실을 예약하며 발생하던 겹침 사고를 데이터 기반 가설 검증과 MVP 전략으로 풀어낸 사례다. 회장단 설문으로 '실시간 현황 확인'이라는 핵심 니즈를 데이터로 확인한 뒤, 기능을 과감히 추려 백엔드 2인 팀으로 한 달 만에 30개 동아리 가입과 노쇼율 60% 감소를 만들어 냈다.

"데이터로 가설을 세우고, 현장의 목소리로 검증하며, 개발팀과의 소통 비용을 직접 줄이는 기획자가 되겠습니다."

비즈니스와 코드 사이

MAU 50만 커머스 스타트업 인턴십에서는 GA4와 사용성 테스트로 이탈 구간을 진단해 A/B 테스트를 설계했다. 하단 구매 바의 가독성 문제를 발견하고 투명도·높이를 조정한 B안은 장바구니 전환율을 12%p 끌어올렸고, 실제 정규 배포로 이어졌다.

스크럼을 이끄는 손

100명 규모 IT 연합 동아리 회장 임기에는 '서로의 언어로 말하기 세미나'와 '크로스오버 해커톤'을 도입해 직무 간 장벽을 낮췄다. 그 결과 데모데이 배포율은 직전 학기 40%에서 85%로 두 배 이상 올랐다.

그가 거듭 강조하는 단어는 '병목'이다. 사용자의 불편과 비즈니스 목표 사이에 끼어 있는 마찰을 정확히 짚고, 가장 적은 자원으로 끊어내는 일 — 그것이 그가 정의하는 PM의 일이다.

Project · 01 · Case File

RoomSync, 동아리방을 다시 짜다

수기 예약과 겹침 사고로 마비된 학내 회의실 50개. 설문으로 니즈를 데이터화한 PM은 MVP 한 장으로 운영을 다시 그렸다.

By FEATURES · 2023.07 — 2023.11 TAGS · MVP기획 · 스크럼리드 · API명세
FIG. 02스크럼 보드 앞의 백엔드 2인과 PM 김태윤. Jira 도입 직후, 학교 동아리방. Photo: 김태윤.

학내 50여 개 동아리가 공용 회의실을 수기로 예약하며 겹침 사고가 빈번했다. 김태윤은 이 비효율을 끊어내기 위해 팀을 꾸렸고, 가장 먼저 한 일은 코딩이 아니라 설문이었다. 구글 폼을 통해 회장단의 페인포인트를 데이터로 변환한 결과, '실시간 예약 현황 확인'이 가장 큰 니즈로 검증됐다.

백엔드 개발자 2명과의 짧은 협업 기간을 고려해 그는 MVP 스코프를 과감히 축소했다. 예약·취소·실시간 뷰만 1차 스프린트에 남기고, 알림·통계 등은 2차로 미뤘다. "MVP 스코프의 80%는 무엇을 뺄지의 결정이었다"는 그의 회고는 이 단계의 본질을 정확히 짚는다.

API 명세를 직접 쓰는 PM

개발 지식을 활용해 API 명세서 초안을 직접 작성, 개발팀의 소통 비용을 줄였다. Jira를 도입해 스크럼 마스터 역할을 수행하며 일정·리스크를 관리했다.

"수기 → 실시간. 그 한 줄을 만들기 위해 우리는 기능 절반을 잘라냈다."

결과

출시 후 한 달 만에 학내 30개 동아리가 가입했고, 노쇼율은 기존 대비 60% 감소했다. 데이터로 가설을 세우고 현장의 목소리로 검증한 PM의 첫 번째 증명 사례다.

Project · 02 · Case File

장바구니 전환율 +12%p 의 비결

MAU 50만 커머스 플랫폼, 45%에 달하던 상세→장바구니 이탈을 사용자 5명의 목소리로 풀었다.

By BUSINESS DESK · 2023.01 — 2023.06 TAGS · GA4 · UT · A/B Test · Nudge

MAU 50만 규모의 커머스 플랫폼에서 김태윤이 맡은 과제는 '장바구니 전환율 개선'이라는 한 줄이었다. 그는 GA4와 앰플리튜드(Amplitude) 데이터를 뜯어보며 상세 페이지에서 장바구니 버튼으로 넘어가는 구간의 이탈률이 45%에 이른다는 사실을 확인했다.

수치만으로는 '왜'를 알 수 없었다. 그는 사용자 5명을 모집해 UT(사용성 테스트)를 진행했고, 하단에 고정된 구매 버튼이 스크롤 시 본문 텍스트를 가린다는 페인포인트를 발견했다.

FIG. 03 — KEY METRIC

이탈률 45%의 구간을 사용성 테스트 5인의 목소리로 풀었고, B안은 정규 배포로 이어졌다.

가설 → A/B 테스트

그는 하단 구매 바의 투명도와 높이를 조정하고, '지금 구매하면 무료배송'이라는 넛지 카피를 추가하는 B안을 기획했다. 가설은 단순했다 — "정보 가림이 사라지면 결정이 빨라진다."

"데이터가 길을 가리키고, 사용자 5명이 그 길의 모양을 그려준다."

결과

B안은 기존 대비 장바구니 담기 전환율을 12%p 상승시켰고, 이 기획안은 실제 정규 배포로 이어졌다. PM의 가설이 프로덕트로 살아남은 첫 번째 사례.

Project · 03 · Case File

100명 동아리, 데모데이 배포율 40 → 85%

기획·디자인·개발 사이의 침묵을 깨기 위해 회장은 두 가지 의식을 도입했다. '서로의 언어로 말하기' 와 '크로스오버 해커톤'.

By CULTURE DESK · 2022.03 — 2022.12 TAGS · 조직문화 · 리더십 · 운영
FIG. 04'서로의 언어로 말하기' 세미나 현장. 기획·디자인·개발 파트가 한 자리에 모인 첫 달, 동아리 라운지.

100명 규모 IT 연합 창업 동아리의 직전 데모데이 배포율은 40%에 그쳤다. 김태윤은 회장직에 오르며 가장 먼저 원인을 찾았고, '직무 간 교류 부재'가 핵심 병목임을 파악했다.

해법은 거창하지 않았다. 한 달에 한 번, '서로의 언어로 말하기 세미나(개발자가 알려주는 깃허브, 디자이너가 알려주는 피그마)'와 '크로스오버 해커톤'을 운영했다. 직무 간 이해도가 오르자 팀원 마찰이 줄어들었다.

결과

연말 데모데이에서 역대 최고치인 85%의 팀이 실제 서비스를 배포했다. 직전 학기 대비 두 배 이상의 결과였다.

"직무 간 통역의 비용을 낮추면, 결과물의 완성도는 두 배가 된다."

그가 PM으로 일하는 방식의 원형은 이 시기 만들어졌다 — 시스템이 아닌 사람의 병목을 푸는 일.
